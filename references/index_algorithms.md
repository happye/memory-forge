# 索引算法详解

## 概述

Memory Forge 使用三层索引结构实现高效检索：

1. **B+树目录索引** (catalog.json) — 按时间排序，支持范围查询
2. **倒排索引** (inverted.json) — 关键词 → 记录ID映射，支持全文检索
3. **标签索引** (tags.json) — 标签 → 记录ID映射，支持分类检索

---

## B+树目录索引

### 为什么选B+树

B+树的核心优势在于**范围查询**。当用户说"查找上周的所有记录"，B+树可以在O(log n + k)时间内返回结果（k为结果数量），而非遍历全部数据。

### 结构设计

```
                [2025-01-15 | 2025-03-01 | 2025-06-01]
               /            |            |            \
    [≤01-15]      [01-15~03-01]  [03-01~06-01]   [≥06-01]
    叶子节点        叶子节点        叶子节点        叶子节点
       ↕              ↕              ↕              ↕
    (链表连接，支持范围遍历)
```

- 阶数 ORDER=32（每个节点最多32个键）
- 所有数据存储在叶子节点
- 叶子节点通过next_leaf指针串联
- 内部节点只存储键和子节点引用

### 查询类型

| 查询 | 复杂度 | 示例 |
|------|--------|------|
| 精确查找 | O(log n) | 某天的记录 |
| 范围查询 | O(log n + k) | 上周的所有记录 |
| 顺序遍历 | O(k) | 最近的N条记录 |

### 持久化

当前实现将B+树展开为扁平的records字典存储在catalog.json中，按mtime排序。未来可扩展为真正的磁盘B+树结构（当记录数超过10万时）。

## 倒排索引

### 原理

倒排索引是搜索引擎的核心数据结构：**从关键词反查包含该关键词的文档**。

```
关键词         记录ID列表
─────────────────────────
"bug"     → [3, 7, 15, 42]
"修复"     → [3, 12, 28]
"认证"     → [7, 15, 33]
```

### 构建过程

1. **分词**：将markdown内容切分为词语
   - 中文：按2+字符的连续CJK序列提取
   - 英文：按2+字母的连续序列提取
2. **去停用词**：过滤高频无意义词（的、了、the、a等）
3. **词频统计**：取每个文件Top 20高频词作为索引词
4. **建索引**：关键词 → 出现该词的记录ID列表

### 查询支持

| 查询 | 实现 |
|------|------|
| AND（交集） | 两个关键词的记录ID列表取交集 |
| OR（并集） | 两个关键词的记录ID列表取并集 |
| NOT（差集） | 从全集中减去某关键词的记录ID列表 |
| 前缀匹配 | 遍历索引键找前缀匹配项 |

### 示例

查询 "认证 bug"（AND模式）：
1. "认证" → [7, 15, 33]
2. "bug" → [3, 7, 15, 42]
3. 交集 → [7, 15]
4. 返回ID为7和15的两条记录

## 标签索引

### 原理

标签索引是倒排索引的特化版本，从markdown中的 `#标签` 语法提取。

```
标签           记录ID列表
─────────────────────────
"安全"     → [7, 15, 33]
"性能"     → [12, 28]
"重构"     → [3, 42]
```

### 优势

- 标签是人工标注的，语义比自动提取的关键词更准确
- 支持标签组合查询（如 #安全 AND #认证）
- 标签权重高于关键词（搜索时 score * 2）

## 综合检索流程

```
用户输入: "查找上周关于认证bug的记录"
              ↓
   ┌──────────────────────────┐
   │ 1. 解析查询              │
   │   关键词: ["认证", "bug"] │
   │   时间: 上周              │
   └──────────┬───────────────┘
              ↓
   ┌──────────────────────────┐
   │ 2. 倒排索引查找关键词     │
   │   "认证" → {7,15,33}     │
   │   "bug"  → {3,7,15,42}   │
   │   AND: {7,15}            │
   └──────────┬───────────────┘
              ↓
   ┌──────────────────────────┐
   │ 3. 时间范围过滤           │
   │   {7,15} ∩ 上周范围       │
   │   → {15}                 │
   └──────────┬───────────────┘
              ↓
   ┌──────────────────────────┐
   │ 4. 返回排序结果           │
   │   按相关度+时间排序       │
   └──────────────────────────┘
```

## 性能参考

| 记录数 | 索引构建 | 精确查询 | 范围查询 |
|--------|---------|---------|---------|
| 100 | <0.1s | <1ms | <1ms |
| 1,000 | <0.5s | <1ms | <2ms |
| 10,000 | <3s | <2ms | <5ms |
| 100,000 | <30s | <5ms | <20ms |

> 注：当前实现为JSON文件全量读写，10万+记录时建议迁移到SQLite或LevelDB。
